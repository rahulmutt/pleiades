# Local eclipse circumstances reference corpus

## Source

Swiss Ephemeris **2.10.03** (via `libswisseph-sys` 0.1.2), Moshier ephemeris
(`SEFLG_MOSEPH`, iflag=4) — no `.se1` data files needed, so `SE_EPHE_PATH` may
point anywhere (a scratch directory such as `/tmp` is sufficient).

Two curated CSVs plus a manifest, generated by the standalone tool
`tools/se-eclipse-local-reference` (a workspace member excluded from the
default build graph — it is not needed to run the gate, only to regenerate
this corpus):

- **`sol-local.csv`** (29 data rows) — solar-eclipse local circumstances.
  For each curated `(eclipse, observer)` case: the topocentric contact
  instants C1..C4 + local maximum (`swe_sol_eclipse_when_loc`), the local
  magnitude/obscuration (`attr[]`), and the Sun's azimuth/altitude at the
  local maximum (`swe_azalt`). Solar contacts are genuinely
  observer-dependent (topocentric), so they come from `when_loc`.
- **`lun-local.csv`** (20 data rows) — lunar-eclipse local circumstances.
  Lunar shadow contacts P1/U1/U2/U3/U4/P4 are **global** instants (the Moon
  enters Earth's shadow at one time for everyone), taken from
  `swe_lun_eclipse_when`; the local layer adds the umbral/penumbral
  magnitude and the Moon's azimuth/altitude + horizon visibility at the
  maximum (`swe_lun_eclipse_how` + `swe_azalt`).
- **`manifest.txt`** — per-CSV row counts + `fnv1a64` checksum of the CSV
  bytes + the SE version string + the conventions above (machine-readable;
  loaded by the Task 11 gate via `include_str!` as a drift guard).

## Reproduction

To regenerate the three files from scratch:

```bash
cd tools/se-eclipse-local-reference
SE_EPHE_PATH=/tmp cargo run -- --out /workspace/crates/pleiades-validate/data/eclipses-local-corpus
```

(`--out <dir>` is the write-mode flag implemented in
`tools/se-eclipse-local-reference/src/main.rs::parse_args`, mirroring
`tools/se-rise-trans-reference`'s write mode; `--dry-run` prints the CSVs +
manifest to stdout without writing, and a bare positional argument is
accepted as a shorthand for `--out`.) Requires libclang + `LIBCLANG_PATH` to
build the `libswisseph-sys` FFI bindings (see the tool's `LICENSE-NOTES.md`);
not required to run the committed gate, which only reads these static files.

The generator is deterministic: no wall-clock time, randomness, or external
network/file dependency (Moshier needs no ephemeris files) enters the
computation, so re-running it produces byte-identical output every time —
confirmed by regenerating to a scratch directory and diffing against the
committed files (see the Task 10 report).

## Anchoring & visibility

`swe_sol_eclipse_when_loc` / lunar `when_loc`-style local search **auto-
advances past eclipses that are not locally visible**, which would make it
impossible to record a below-horizon / Moon-down observer against the
*intended* eclipse. So each case is first anchored to a specific eclipse via
a **global** search seeded a few days before the event, then characterised
locally:

- **Solar**: `swe_sol_eclipse_when_glob` finds the next global eclipse after
  the seed date. `swe_sol_eclipse_when_loc` is then run from the same seed.
  If its local maximum lands within 2 days of the global maximum
  (`SAME_ECLIPSE_TOL_DAYS`), the eclipse is locally visible → topocentric
  C1..C4 + `attr` come from `when_loc`, `se_any_visible=1`. Otherwise the Sun
  is down / no eclipse is visible there → magnitude and az/alt come from
  `swe_sol_eclipse_how` evaluated **at the global maximum**, the four
  contact columns are left empty, `se_any_visible=0`.
- **Lunar**: `swe_lun_eclipse_when` gives the global shadow contacts
  (identical for every observer, matching the engine's geocentric contact
  model). `swe_lun_eclipse_how`, evaluated at that same global maximum,
  gives the per-observer umbral/penumbral magnitude and the Moon's apparent
  altitude; `se_any_visible = (apparent altitude > 0)`.

## Time base (UT, not TDB — same posture as `rise-trans-corpus`)

All instants in both CSVs are **UT Julian days**, because the SE eclipse
functions (`swe_sol_eclipse_when_glob/when_loc/how`,
`swe_lun_eclipse_when/how`) natively return UT, not TT/TDB. This mirrors
`tools/se-rise-trans-reference` / `rise-trans-corpus`'s ΔT posture: rather
than convert SE's UT output to TDB with `swe_deltat` (which would inject an
independent ΔT model into the reference data), the corpus stays in UT and
the Task 11 gate converts the *engine's* native-TDB `LocalCircumstances`
instants to UT for comparison — exactly as `validate-rise-trans` compares
against the SE `_ut` columns rather than converting the fixture to TDB. This
keeps the reference data a direct, unmodified transcript of what SE
reported, with exactly one documented ΔT crossing (in the gate, not in the
fixture).

Every emitted maximum/contact instant is asserted (via a Rust `assert!` in
the generator) to fall inside the shared **1900-01-01 .. 2100-01-01** window
(JD 2 415 020.5 .. 2 488 069.5), matching the engine's hard
`EclipseError::OutOfWindow` clamp.

## Azimuth convention

`swe_azalt` returns azimuth measured from **SOUTH, increasing WESTWARD**, in
`[0, 360)` degrees (SE native convention). The `pleiades-eclipse` local layer
matches this convention exactly — do not reinterpret the `se_max_az_deg`
columns as from-north/clockwise. Altitude columns are **true** altitude and
**apparent** (refraction-corrected, standard atmosphere 1013.25 hPa / 15°C)
altitude in degrees, both computed via `swe_azalt` from geocentric
apparent ecliptic-of-date Sun/Moon positions (`swe_calc_ut`,
`SEFLG_MOSEPH`).

## Curated case list

**Solar** (`sol-local.csv`, 29 rows) — four historical eclipses, each with
observers spanning in-path (total/annular/hybrid), partial-visible, and
below-horizon/no-eclipse:

- **2017-08-21** "Great American" total eclipse (10 observers: Salem OR,
  Madras OR, Idaho Falls, Kansas City, Nashville, Charleston SC in-path;
  NYC, Los Angeles, Miami partial; Tokyo below-horizon).
- **2024-04-08** total eclipse (8 observers: Mazatlán, Torreón, Dallas,
  Indianapolis, Cleveland in-path; Miami, NYC partial; Tokyo
  below-horizon).
- **2023-10-14** annular eclipse (6 observers: San Antonio, Albuquerque,
  Corpus Christi in-path; Denver, NYC, Mexico City partial).
- **2013-11-03** hybrid eclipse (5 observers: Libreville, mid-Atlantic
  in-path; Madrid, Lagos, Nairobi partial).

**Lunar** (`lun-local.csv`, 20 rows) — three historical eclipses, each with
observers spanning Moon-up (visible) and Moon-down (below horizon) at the
maximum:

- **2018-07-27** total lunar eclipse (8 observers: Cape Town, Rome, Delhi,
  Nairobi, Moscow, Sydney visible; Los Angeles, NYC Moon-down).
- **2019-07-16** partial lunar eclipse (6 observers: Rome, Cairo,
  Johannesburg, London, Delhi visible; Los Angeles Moon-down).
- **2020-01-10** penumbral lunar eclipse (6 observers: Delhi, Moscow,
  Beijing, Rome, London visible; NYC Moon-down).

Each case seeds the global/local search a few days before its eclipse
(`(year, month, day)` in the generator's `Case` tuples); labels encode the
eclipse (year + type code) and the observer city, with a `_partial` /
`_belowhoriz` / `_down` suffix where the observer does not see the full
local event.

## Columns

### `sol-local.csv` (16 columns)

```
label,lat_deg,lon_deg,elev_m,se_max_jd_ut,se_c1_jd_ut,se_c2_jd_ut,se_c3_jd_ut,se_c4_jd_ut,se_local_type,se_magnitude,se_obscuration,se_max_az_deg,se_max_true_alt_deg,se_max_app_alt_deg,se_any_visible
```

- **label**: case identifier (eclipse + observer city, see above).
- **lat_deg / lon_deg / elev_m**: observer geodetic latitude (+N), longitude
  (+E), height above the ellipsoid in meters — the exact inputs handed to
  `swe_sol_eclipse_when_loc` / `swe_azalt`.
- **se_max_jd_ut**: local maximum instant (`when_loc`) if the eclipse is
  locally visible, else the global maximum instant (`when_glob`). UT
  Julian day.
- **se_c1_jd_ut .. se_c4_jd_ut**: topocentric first/second/third/fourth
  contact instants (UT Julian day). Empty when absent: C2/C3 are absent
  for a partial-only local view (no annular/total phase), and all four are
  empty when the eclipse is not locally visible (no topocentric contacts
  exist to report).
- **se_local_type**: the SE return flag masked to
  `SE_ECL_ALLTYPES_SOLAR` (`4`=total, `8`=annular, `16`=partial,
  `32`=annular-total, `0`=no eclipse visible here).
- **se_magnitude**: eclipse magnitude, the diameter fraction (`attr[0]`).
- **se_obscuration**: eclipse obscuration, the area fraction (`attr[2]`).
- **se_max_az_deg**: Sun's azimuth at the maximum, from SOUTH increasing
  WESTWARD, `[0, 360)` degrees.
- **se_max_true_alt_deg / se_max_app_alt_deg**: Sun's true and apparent
  (refracted) altitude at the maximum, in degrees.
- **se_any_visible**: `1` if the eclipse is above the horizon and locally
  visible at its maximum, else `0` (Sun down, or no eclipse visible at all
  from this location).

### `lun-local.csv` (18 columns)

```
label,lat_deg,lon_deg,elev_m,se_max_jd_ut,se_p1_jd_ut,se_u1_jd_ut,se_u2_jd_ut,se_u3_jd_ut,se_u4_jd_ut,se_p4_jd_ut,se_type,se_umbral_mag,se_penumbral_mag,se_max_az_deg,se_max_true_alt_deg,se_max_app_alt_deg,se_any_visible
```

- **label / lat_deg / lon_deg / elev_m**: as above.
- **se_max_jd_ut**: global maximum instant (`swe_lun_eclipse_when`), UT
  Julian day — identical for every observer of the same eclipse.
- **se_p1_jd_ut / se_u1_jd_ut / se_u2_jd_ut / se_u3_jd_ut / se_u4_jd_ut /
  se_p4_jd_ut**: global penumbral-begin, umbral-partial-begin,
  umbral-total-begin, umbral-total-end, umbral-partial-end,
  penumbral-end instants (UT Julian day). Empty when the corresponding
  phase does not occur: U1..U4 absent for a penumbral-only eclipse, U2/U3
  absent for a partial-only eclipse.
- **se_type**: the SE return flag masked to `SE_ECL_ALLTYPES_LUNAR`
  (`4`=total, `16`=partial, `64`=penumbral).
- **se_umbral_mag / se_penumbral_mag**: umbral and penumbral magnitude at
  the maximum (`attr[0]`, `attr[1]` from `swe_lun_eclipse_how`).
- **se_max_az_deg / se_max_true_alt_deg / se_max_app_alt_deg**: Moon's
  azimuth (from SOUTH, increasing WESTWARD) and true/apparent altitude at
  the maximum, in degrees.
- **se_any_visible**: `1` if the Moon's apparent altitude is `> 0` at the
  maximum (Moon above the horizon), else `0`.

## Checksum / drift-guard scheme

`manifest.txt` records, per CSV, `rows=<N>` and
`checksum=<fnv1a64 digest>` — the 64-bit FNV-1a hash (offset basis
`0xcbf29ce484222325`, prime `0x100000001b3`) of the exact bytes written to
disk. This is byte-for-byte the same scheme as `pleiades_apparent::fnv1a64`
and every other `se-*-reference` manifest in this repo (e.g.
`rise-trans-corpus/manifest.txt`). The Task 11 gate loads both CSVs and
`manifest.txt` via `include_str!` at compile time and recomputes the digest
of each CSV, failing closed if it does not match the recorded value — this
catches any accidental hand-edit or partial regeneration of the committed
corpus before a single numeric comparison runs.
