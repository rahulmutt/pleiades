# Fictitious (hypothetical) body reference corpus

## Source

Swiss Ephemeris **2.10.03** (via `libswisseph-sys` 0.1.2), Moshier ephemeris
(`SEFLG_MOSEPH`, iflag=4) — no `.se1` data files needed for the ephemeris
itself, though the fictitious-body **element** file is required (see below).

One curated CSV plus a manifest, generated by the standalone tool
`tools/se-fictitious-reference` (a workspace member excluded from the default
build graph — it is not needed to run the gate, only to regenerate this
corpus):

- **`fictitious.csv`** (570 data rows: 19 bodies × 30 samples) — for each of
  SE's fictitious bodies 40..=58, a per-body date grid sampled evenly
  (inclusive) across 1900-2100 TT: geocentric J2000-ecliptic
  longitude/latitude/distance from `swe_calc`.
- **`manifest.txt`** — row count + `fnv1a64` checksum of the CSV bytes + the
  SE version string + the flags/window conventions above (machine-readable;
  loaded by the Task 8 gate via `include_str!` as a drift guard).

### Flags (GEOMETRIC, not apparent)

`SEFLG_MOSEPH | SEFLG_J2000 | SEFLG_TRUEPOS | SEFLG_NOABERR | SEFLG_NOGDEFL`
= `4 | 32 | 16 | 1024 | 512` = `1588`. Bare `SEFLG_J2000` alone would emit
*apparent* positions (light-time + aberration + gravitational deflection),
which would NOT match `FictitiousBackend::position`'s geometric
geocentric-J2000 boundary: `SEFLG_TRUEPOS` drops light-time, `SEFLG_NOABERR`
drops annual aberration, `SEFLG_NOGDEFL` drops gravitational light
deflection.

### Elements file

SE's built-in fallback element set only covers bodies 40-54; bodies 55-58
(Vulcan, White Moon, Proserpina, Waldemath) require the `seorbel.txt` element
file (`SE_FICT_OFFSET_1`-based bodies beyond the built-in table return
`SE_NFICT_ELEM` errors otherwise). The generator commits the verbatim
upstream `seorbel.txt` under `tools/se-fictitious-reference/data/` and points
`swe_set_ephe_path` at that directory.

## Reproduction

To regenerate the two files from scratch:

```bash
cargo run --manifest-path tools/se-fictitious-reference/Cargo.toml -- \
    --out crates/pleiades-validate/data/fictitious-corpus
```

(`--out <dir>` is the write-mode flag; `--dry-run` prints the CSV + manifest
to stdout without writing; `--ephe <dir>` overrides the `seorbel.txt`
directory, defaulting to the tool's own committed `data/`.) Requires libclang
+ `LIBCLANG_PATH` to build the `libswisseph-sys` FFI bindings; not required to
run the committed gate, which only reads these static files.

The generator is deterministic: no wall-clock time, randomness, or external
network dependency enters the computation (Moshier needs no ephemeris data
files, and the fictitious elements are read from the committed `seorbel.txt`),
so re-running it produces byte-identical output every time.

## Time base

All instants are **TT (Terrestrial Time) Julian days**, matching
`FictitiousBackend::position`'s accepted time scales (`TimeScale::Tt` /
`TimeScale::Tdb`, TT≈TDB) — the Task 8 gate feeds each row's `jd_tt` directly
to the backend with no ΔT crossing.

Every sample falls inside the shared **1900-01-01 .. 2100-01-01** window (JD
2 415 020.5 .. 2 488 069.5 TT), matching the sibling `se-eclipse-local-reference`
/ `se-eclipse-global-reference` window and the packaged ephemeris data's own
covered range.

## Sampling

19 bodies (SE numbers 40..=58, `SE_FICT_OFFSET_1` (39) + 1..=19) × 30 samples
per body, evenly spaced **inclusive** of both window endpoints (i.e. the
first and last sample of every body sit exactly on 1900-01-01 and
2100-01-01). Bodies, in SE-number order:

Cupido (40), Hades (41), Zeus (42), Kronos (43), Apollon (44), Admetos (45),
Vulkanus (46), Poseidon (47), Transpluto/Isis (48), Nibiru (49), Harrington
(50), NeptuneLeverrier (51), NeptuneAdams (52), PlutoLowell (53),
PlutoPickering (54), Vulcan (55), WhiteMoon/Selena (56), Proserpina (57),
Waldemath (58).

## Columns (6)

```
label,se_body,jd_tt,lon_deg,lat_deg,dist_au
```

- **label**: `CelestialBody` enum-style variant name (`Cupido`, `Vulkanus`,
  `NeptuneLeverrier`, `WhiteMoon`, …) — identical spelling to the
  `pleiades_fict::elements` table and to `CelestialBody`'s variant names, so
  the gate can key on either this column or `se_body` (the gate keys on
  `se_body`, the SE integer, for a simpler match arm).
- **se_body**: SE body number, `40..=58`.
- **jd_tt**: sample instant, TT Julian day.
- **lon_deg / lat_deg**: geocentric ecliptic longitude/latitude, J2000 mean
  ecliptic, degrees. GEOMETRIC (no light-time/aberration/deflection).
- **dist_au**: geocentric distance, AU.

## Checksum / drift-guard scheme

`manifest.txt` records `rows=<N>` and `checksum=<fnv1a64 digest>` — the
64-bit FNV-1a hash (offset basis `0xcbf29ce484222325`, prime `0x100000001b3`)
of the exact CSV bytes written to disk. This is byte-for-byte the same scheme
as `pleiades_apparent::fnv1a64` and every other `se-*-reference` manifest in
this repo. The Task 8 gate loads the CSV and `manifest.txt` via
`include_str!` at compile time and recomputes the digest, failing closed if
it does not match the recorded value.

## Measured SE parity (Task 8) and the Nibiru carve-out

Every one of the 18 non-Nibiru bodies matches this corpus to a small
fraction of an arcsecond in longitude/latitude and a few microAU in distance
— see `crates/pleiades-validate/src/fictitious_thresholds.rs` for the exact
measured maxima and the resulting gate ceilings (`LONGITUDE_ARCSEC`,
`LATITUDE_ARCSEC`, `DISTANCE_AU`, each ~1.4× the observed max over those 18
bodies).

**Nibiru** (SE 49) is a documented, expected outlier: its `seorbel.txt`
equinox is ~370 AD — about 1630 years before J2000 — well outside the most
accurate range of the IAU-1976 precession helper
(`pleiades_fict::frame::rotate_ecliptic_to_j2000`) used to carry every
fictitious body's elements to the J2000 mean ecliptic. Its residual (~1.26″
longitude, ~0.71″ latitude, ~6.1e-6 AU distance) is still small in absolute
terms but measurably larger than the other 18 bodies (roughly 2.7× the worst
non-Nibiru longitude residual, ~12× the worst latitude residual). Rather than
inflate the global ceilings above to swallow this one body, Nibiru is gated
under its own wider per-body ceilings (`NIBIRU_LONGITUDE_ARCSEC`,
`NIBIRU_LATITUDE_ARCSEC`, `NIBIRU_DISTANCE_AU`, ~1.4× Nibiru's own measured
maxima) in `validate_fictitious_corpus`. This is a known, explained
limitation of the fictitious-body model for bodies with far-from-J2000
equinoxes, not a defect.

## A boundary interaction fixed during Task 8

The corpus samples **inclusive** of both window endpoints, so the first and
last sample of every body sit exactly on the packaged ephemeris data's own
covered-range boundary (1900-01-01 / 2100-01-01 TT). `FictitiousBackend`'s
motion (finite-difference speed) computation originally took a symmetric
±0.5-day probe around the requested instant, which — for a heliocentric
fictitious body sampled at either endpoint — needs the Sun's position half a
day *beyond* that boundary, and a bounded backend correctly rejects that,
failing the whole `position()` call over a motion-only edge effect. This is
fixed in `pleiades-fict::backend::FictitiousBackend::motion` by falling back
to a one-sided difference against the body's own already-computed position
when one side of the symmetric probe is unavailable; the position itself was
never affected, only the boundary rows' speed estimate degrades from central
to one-sided difference (see the regression test
`backend::tests::motion_falls_back_to_one_sided_difference_at_a_data_window_edge`).
