# Phase 1 — Artifact Accuracy and Packaged-Data Production

## Goal

Ship a production-quality compressed ephemeris artifact for 1500-2500 CE that satisfies `spec/data-compression.md`, `requirements.md` FR-9, and the packaged-backend responsibilities in `spec/backends.md`.

## Starting point

The codec, artifact structures, deterministic regeneration path, checksums, generation manifest, output-support profile, body-class cadence summary, fit margin summary, bundled-body cadence summary, channel fit-outlier reports, boundary checks, lookup benchmarks, batch-lookup benchmarks, and decode benchmarks already exist. The packaged artifact profile validation now fails closed if any built-in output would remain unlisted, so profile drift cannot silently omit a requested output class. The phase-2 corpus alignment summary now also has its own dedicated validation error type, keeping direct corpus-alignment failures distinct from target-threshold drift in the public API. The generator now applies a six-point Chebyshev-Lobatto fit plus measured-fit comparison against fallback reconstruction on short spans, can attach residual-correction channels across the bundled body set when they improve the measured fit, now explores residual-channel combinations rather than only greedy single-channel additions and now also explores all remaining channel-order permutations when composing those channels, and now uses channel-specific residual sample fractions so dense angular bodies can keep the dense lattice while distance residuals stay sparse; it also uses a denser residual sample lattice for luminaries and custom bodies so the Moon-adjacent and long-window custom-body correction search is body-specific instead of flat; the checked-in fixture is regenerated from that path. The distance channel path now also falls back to a cubic four-point control-point reconstruction before quadratic fallback when the higher-order fit does not quantize cleanly. Fit-outlier diagnostics now use a denser report lattice for body/channel summaries without changing the calibrated threshold lattice, and the generator now applies body-specific validation/outlier sampling for luminaries, lunar points, selected asteroids, Pluto, and custom bodies. The current generator also gives luminaries, lunar points, Pluto, selected asteroids, and custom bodies a denser fit-candidate lattice before the legacy fallback ladder on those sensitive spans, now including 10-point, 12-point, 14-point, 16-point, 18-point, and 20-point Chebyshev-Lobatto options for the dense bodies, and the generator now also tries 8-point and 10-point Chebyshev-Lobatto baseline candidates for every body before that body-specific dense ladder, with inner and outer planets now also using 12-point and 14-point medium candidate baselines before fallback, and the best dense candidate now wins before fallback. Fit-candidate scoring now prefers the smaller sample count on equal error, keeping identical-fit candidates compression-lean. The scorer now also breaks equal-error, equal-sample-count ties by preferring the simpler segment. The normalized-intermediate summary now carries a deterministic checksum, routes through a validated wrapper in release-facing validation surfaces, and is cross-checked against current rendering during release-bundle verification, and the target-threshold posture is now modeled as a typed production-ready state so the finalized production-threshold policy stays explicit instead of being hidden in raw text; the packaged-artifact target-threshold summary now also routes through a validated wrapper in release-facing validation/report surfaces, so the target-threshold line itself fails closed before report rendering. The storage/reconstruction and production-profile summary surfaces now route through explicit validated wrappers before rendering, so release-facing artifact posture lines fail closed when summary validation drifts, and the output-support posture now has validated wrappers for the full capability line and explicit support-entry rendering, while the fit margin summary now has a validated structured wrapper for release-facing error-statistics reuse. The validation and artifact smoke-report paths now cache expensive report objects and use reduced timing subsets for tractable bundle verification under the test harness, the packaged-artifact body cadence summary now has a validated structured wrapper and fail-closed report rendering, and the release-bundle verifier now also cross-checks it against the live renderer, the release-bundle verifier now also cross-checks the packaged-artifact target-threshold summary against the current render helper, and the regeneration helper now caches the deterministic rebuilt artifact in-process so repeated validation and report invocations avoid rebuilding the full artifact, and the write path now reuses the committed artifact bytes while `--check` still exercises regeneration validation so fixture-writing smoke tests stay tractable. The CLI regeneration path now also uses the cached regenerated bytes helper directly, so repeated sidecar writes stay aligned with the live rebuild cache instead of the committed fixture bytes. The CLI regression suite now also invokes the sidecar-writing path twice and checks the emitted artifact and sidecars stay stable across repeated writes. The packaged-artifact generation manifest checksum summary and the production-generation manifest summary/checksum helpers now cache their rendered strings in-process, trimming repeated bundle/report recomputation.

## Implementation goals

- Rework fitting and reconstruction until body/channel errors meet published thresholds for the advertised scope.
- Use validated public inputs from Phase 2 rather than ad hoc sparse samples for production fitting.
- Keep the finalized body-class/channel thresholds synchronized with source-fit and independent hold-out validation.
- Decide whether polynomial order, Chebyshev fitting, residual tables, per-body cadence, or channel-specific storage changes are required.
- Residual correction now uses channel-specific sample fractions so angular channels can keep the dense lattice while distance channels stay on the sparse lattice, and the residual-channel lattice helper now explicitly centralizes the dense-body angular-channel rule while the regression coverage now includes selected asteroids and custom bodies.
- Residual correction search now also explores all remaining channel-order permutations when composing channels, so order-sensitive candidates no longer depend on the previous suffix-only traversal, and equal-error candidates now prefer the smaller residual footprint. Equal-error candidate-versus-fallback decisions now prefer the simpler segment, and the candidate-vs-fallback path is now accuracy-first so a worse fit no longer wins purely because it is simpler.
- Higher-order fit candidates now share a control-point fallback helper across longitude, latitude, and distance channels, so fit reconstruction can degrade to a lower-order interpolation when quantization would otherwise overflow, and the fallback path now also tries a dense quarter-point control-point lattice before the exact-third fallback on irregular spans, with one-sixth/five-sixth probes now available on very long dense spans before the one-third/two-thirds fallback; when those sixth-point probes are unavailable, the split search still falls through to the one-third/two-thirds ladder instead of reverting directly to midpoint splitting. The release-facing generation-policy prose now also names the quarter-biased dense-body split posture alongside those probe ladders.
- Completed: validation now rejects any unlisted artifact output classification, and release-facing output-support summaries render the explicit empty unlisted bucket.
- Preserve deterministic generation parameters, normalized-intermediate checksums, artifact checksums, and encoded-size accounting.
- Completed: the comparison body-class tolerance summary renderer now validates each class row before formatting, so malformed tolerance rows fail closed instead of leaking partial drift text.
- Completed: the packaged-artifact target-threshold scope envelopes summary now has a typed validated wrapper, and the release-facing scope-envelope helper now reuses that structured posture instead of rebuilding the lines manually.
- Completed: the packaged-artifact target-threshold state summary now also has a semantic-drift release-bundle regression, so the release-facing state line fails closed even after checksum refreshes.
- Completed: the packaged-artifact body-class span-cap summary now exposes a direct validated helper, and the report surfaces now reuse it instead of duplicating the validation step before rendering.
- Completed: the release bundle verification tests now also cover missing packaged-artifact normalized intermediate summary entries, so that artifact sidecar cannot silently disappear from the staged bundle.
- Make artifact validation fail on threshold violations, capability drift, checksum drift, malformed manifests, or unsupported request shapes.
- Keep performance benchmarks visible, but treat accuracy, reproducibility, and report tractability as release blockers.

## Completion criteria

Phase 1 is complete when a clean checkout can regenerate the packaged artifact and pass published reference/hold-out thresholds for the advertised bodies, channels, request modes, and 1500-2500 CE range.

## Out of scope

- Broadening body claims without Phase 2/3 evidence.
- Implementing apparent, topocentric, UTC/Delta-T, or native-sidereal behavior.
- Promoting catalog compatibility claims.
