- Doc drift: `docs/release-process.md` says "14 library crates"; there are now

## Decisions

- **Path:** automated release-plz flow (documented primary path), not the
  manual `cargo-release` fallback. Automation is healthy â€” the PR job works
  and the release-job failure disappears with the lockstep bump.
- **Toolchain bump:** land rust 1.97.1 (mise.toml + workspace `rust-version`)
  on `main` *before* merging the Release PR, so 0.4.0 ships with the current
  patch toolchain. 1.97.1 is a patch release; MSRV impact negligible.
- **Housekeeping in scope:** release-process.md drift fix (pre-release),
  PLAN.md status refresh + post-publish verification (post-release).
- **Chore commits go directly to `main`**, matching the repo's existing
  pattern for toolchain bumps (`bump rust`, `bump release-plz`).

## Error handling

- **Publish fails midway:** re-run the release job â€” crates already published
  at 0.4.0 are skipped (documented recovery path).
- **Audit failure after the toolchain bump:** fix before any push.
- **Changelog problems:** push fixes to the release branch; never hand-edit
  versions.
- **Bad release:** fix forward with a patch release; crates.io never allows
  re-publishing a version.
